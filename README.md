## 工单系统：怎么提、注意什么

DMIT 的售后支持唯一渠道是工单系统（Ticket），没有实时在线客服。登录后台，在"技术支援"下找到"建立服务单"，填写主题和问题描述即可。

几个实际使用里要注意的点：

1. **工单用英文写**。虽然你用中文提交大概率也看得懂，但官方的回复是英文，如果有术语不懂可以直接机翻。
2. **同时只能开一个工单**。如果你试图新开第二个，系统会提示"您已有未结服务单"，要先关掉第一个才能开新的。
3. **不要重复催单**。DMIT 工单按最后更新时间排队——你每催一次，相当于把自己的工单拉到队尾重新排。等着就好。
4. **工单提交后会有邮件通知**。回复之后你的邮箱会收到提醒，不用一直盯着后台刷新。

**响应时间**：正常情况下 24 小时以内，官方承诺最长不超过 72 小时。节假日会慢一些。用过的人普遍反馈是 6 小时以内就有回复，有时候快的话几十分钟就回了。技术类问题（需要查日志、排查路由的那种）可能会更慢，这个没有保证时效。

## 退款政策：说清楚，不踩雷

这块是 DMIT 售后里被问得最多的，我把官方政策整理了一遍，直接说规则。

**全额退款条件（扣除支付网关手续费）：**
- 购买不超过 3 天
- 流量使用不超过 30GB
- 是同一产品系列的首次（或未满 3 次）退款申请

**部分退款条件：**
- 购买 30 天以内
- 退款金额按实际支付额计算，取"剩余流量价值"和"剩余使用时间"两者中较低的

**退款到哪里：**
- 退到 DMIT 账户余额：不扣手续费，直接能用来续费或买新套餐
- 退到原支付方式：会扣支付网关手续费（支付宝例外，退款时手续费会返还）

**退款处理时效**：通常 48 小时内处理，高峰期可能延迟。

**不予退款的情况（重点）**：
- 同一产品系列退款已达 3 次
- 遭受 DDoS 攻击期间
- 以网络不好为由（这条很关键，讨厌但确实写在 TOS 里）
- IP 地理位置原因
- IP 在部分地区不可用但流量使用已超 3GB
- 单方面向支付公司发起争议

讲真，退款政策这块比很多小商家透明不少。3 天内不满意可以退，30 天内按比例退，这个兜底对新用户来说试错成本低。

**退款怎么申请**：登录后台，提交工单给 Sales Department，主题写 "REFUND REQUEST"，说明原因（注意不能写"网络不好"这类理由）。

## DMIT 售后的几个真实细节

聊了这么多政策，说几个实际用下来的感受。

工单这件事，DMIT 不算是那种可以聊天的客服，他们是非托管服务——你的服务器出了基础配置以外的问题，比如 SSH 连不上、系统崩了、某个端口被封，提工单基本都能解决，而且回复不是那种套话式的"我们已收到您的反馈"，是真的在看你的问题。

碰到路由问题或 DDoS 期间，那就不一样了。这期间工单可能等很久，而且如果是 DDoS 攻击导致的网络质量下降，退款是不受理的。这个政策对部分人来说不太友好，但它白纸黑字写在 TOS 里，买之前知道就行。

还有一点：DMIT 售后是纯英文回复，但工单本身可以用中文写，对方会看懂的。语言上不是障碍，顶多是回复需要稍微翻译一下。

顺便提一嘴：DMIT 在硬件升级上有个良心做法——定期升级宿主机时，已有套餐会同步升级配置，不加价。所以你今天买的方案，半年后可能硬件规格就悄悄变好了。这种事在 VPS 行业真的不多见。

## 常见问题（FAQ）

**Q：DMIT 工单可以用中文提交吗？**
可以。工单支持中文提交，客服能看懂，但回复是英文。

**Q：退款申请提交后多久到账？**
退款处理通常在 48 小时以内，退到原支付方式的到账时间取决于支付渠道，支付宝通常较快。

**Q：网络质量不好可以申请退款吗？**
不可以。官方 TOS 明确写明"网络状况不佳"不是退款理由。如果你不确定线路适不适合自己，可以先用测试 IP ping 一下延迟，或者买入门款先跑 3 天，不满意在流量 30GB 以内退。

**Q：IP 被墙了一定要提工单换吗？**
洛杉矶 Pro 和 EB 系列用户现在可以自助换 IP，不用等工单。香港和东京系列目前还需要走工单流程。

**Q：流量用完后会停机吗？**
不会直接停机。DMIT 的做法是超量后限速（降到 100Mbps–1Gbps 不等，具体看套餐），不中断服务。这个设计对跑长期任务的用户来说很友好。

**Q：买 DMIT 适合我吗？**
如果你需要对中国大陆访问稳定的 VPS，预算不算特别紧，不想三天两头折腾换商家——DMIT 很合适。如果你只是想找个便宜跑脚本的机器，它的价格不太划算，可以考虑它的 T1 国际线路系列作为入门。
